# Store Model

Проект "Store Model" — это основа для e-commerce-приложения, реализованная с использованием объектно-ориентированного подхода (ООП).
В рамках проекта описаны основные сущности, такие как продукты и категории, с поддержкой статистики по созданным объектам.

### Возможности 
В рамках проекта реализовано:
- Класс `Product` — описание товаров с поддержкой:
  - приватного атрибута цены `(__price)`, 
  - геттеров и сеттеров с валидацией, 
  - `@classmethod new_product()` для создания продукта из словаря с логикой слияния дубликатов.
  - переопределённый метод `__str__`
  - переопределённый метод `__add__` — сумма стоимости всех товаров `(цена × количество)`
    - Сложение доступно только между объектами одного класса (например, два смартфона).
    - При попытке сложить несовместимые типы (например, смартфон и траву) выбрасывается `TypeError`.


- Класс `Category` — категории товаров с поддержкой:
  - приватного списка товаров `(__products)`, 
  - метода `add_product()` с автоматическим увеличением счётчика товаров
    - Разрешено добавлять только объекты `Product` или его наследников.
    - Попытка добавить неподходящий объект вызовет `TypeError`.
  - геттера products — возвращает отформатированный список товаров.
  - переопределённый `__str__` — строковое представление категории
  - поддержка итерации по продуктам категории с помощью кастомного итератора  

  
- Загрузка данных из JSON-файла `(load_data_from_json())`, автоматическое создание объектов `Product` и `Category`.


- класс-наследника `Product`. `LawnGrass`
  - `LawnGrass` — расширяет товар атрибутами: страна-производитель, срок прорастания, цвет.


- класс-наследника `Product`. `Smartphone`
  - `Smartphone` — расширяет товар атрибутами: производительность, модель, память, цвет.


- Базовый абстрактный класс `BaseProduct`
  - Вынесена общая логика: атрибуты, геттеры/сеттеры, метод `add_product()`
  - Наследуется классами `Product`, `Smartphone`, `LawnGrass`


- Класс-миксин `InitMessageMixin`
  - При создании объекта печатает в консоль информацию о классе и переданных параметрах
  - Используется в `Product` (и, следовательно, в `Smartphone`, `LawnGrass`)

## Добавленные модули 
### модуль `product.py`
- `class Product` - Класс для представления товара в магазине.

### модуль `category.py`
- `class Category` - Класс для представления категории товаров.

### модуль `loaders.py`
- `load_data_from_json` - Загружает категории и товары из JSON-файла, возвращает список Category.

### модуль `smartphone.py`
- класс-наследника `Product`. `Smartphone` — расширяет товар атрибутами: производительность, модель, память, цвет.

### модуль `LawnGrass.py`
- класс-наследника `Product`. `LawnGrass` — расширяет товар атрибутами: страна-производитель, срок прорастания, цвет.

### модуль `base_product.py`
- `class BaseProduct` — абстрактный базовый класс для всех товаров. Вынесена общая логика: атрибуты, геттеры/сеттеры, метод `add_product`. Наследуется в `Product`.

### модуль `mixins.py`
- `class InitMessageMixin` — миксин, печатающий информацию при создании объекта. Подключается к `Product` и используется во всех его потомках.
